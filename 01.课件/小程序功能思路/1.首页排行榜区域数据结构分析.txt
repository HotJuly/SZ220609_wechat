数据结构:
[
	{
		id:100,
		name:"国电榜",
		list:[
			{
				imgUrl:"xxx.jpg",
				name:"水之幻想",
				id:1001
			}
		]
	}
]




思路分析过程:
	1.现在具有一个swiper组件,内部存储这多个swiper-item,每个swiper-item都是一个榜单
		由于存储的内容都是榜单,含义相近,所以选择使用数组

	2.每个swiper-item都是用于显示一个榜单的数据,每个榜单都具有自己的id,榜单名称以及榜单列表
		由于存储的内容是榜单不同方面的数据,含义不相近,但是描述的是同一个榜单,所以选择使用对象

	3.每个榜单中都都具有榜单列表,列表中展示的是多首歌曲
		由于存储的内容都是歌曲,含义相近,所以选择使用数组

	4.每个歌曲都具有自己的图片,排名,歌曲名称以及歌曲id
		所以选择使用对象存储


存储数据的类型:
	1.对象
		对象存储数据是以键值对形式存储数据的,也就是key:value
		那么也就是说,假设对象需要存储3个数据,需要给每个数据都准备一个唯一的标识

		如果现在具有一堆数据,这些数据的含义不相同,但是都是某个东西的不同方面的数据,那么就选择使用对象

	2.数组
		数组存储数据是以下标形式存储数据的,也就是index:value
		那么也就是说,假设数组需要存储3个数据,不需要准备任何的唯一标识,只需要按顺序排列即可
		
		如果现在具有一堆数据,这些数据的含义相近,那么就选择使用数组存储
